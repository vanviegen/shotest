# UI Testing Guidance

After running `npx playwright test`, results for a test are in its `test-results/<TEST_NAME>/` directory. It may contain:

- <POSITION>.png: screenshot (you can view this image for visual/layout verification)
- <POSITION>.body.html: DOM body snapshot (useful for debugging and for writing test selectors)
- <POSITION>.head.html: DOM head snapshot
- error.txt: error message + stack trace (in case of an error)
- manifest.json: metadata for 'npx shotest' review tool (for human only; you may want to instruct your human to do this when you're presenting new features)

TEST_NAME is generated by Playwright based on the spec file name and the name of the test.
POSITION consists of the line number in the test file, suffixed by a letter for uniqueness. (Eg: `0010a`.)

When you create new UI features:
1. Make sure they are tested by `tests/*.spec.ts` Playwright test files.
2. Run the tests.
3. If you are capable of viewing images, review images at some carefully selected points in the test flow, in order for you to see your new/changed features.
4. Fix any issues (test or visual; humans like things to look good!) and repeat from 1 until satisfaction.
